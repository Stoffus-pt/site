Publicar o site Stoffus online
==============================

Dois ambientes
--------------
1) Pré-visualização (testes): GitHub Pages
   https://stoffus-pt.github.io/site/
   Actualiza ~2 min após cada push para main.
   Sem PHP/CMS — só HTML/CSS/JS/assets.

2) Produção (site oficial): alojamento stoffus.pt
   CMS PHP, formulários, uploads.
   É este o URL “público” para clientes.


FASE DE TESTES — GitHub Pages
-----------------------------
Cada push para main actualiza automaticamente a pré-visualização:

  https://stoffus-pt.github.io/site/

1. GitHub → Stoffus-pt/site → Settings → Pages
2. Source: GitHub Actions
3. Push para main (ou Actions → "Pré-visualização (GitHub Pages)" → Run)


PRODUÇÃO — stoffus.pt (FTP)
---------------------------
O workflow "Publicar site (FTP)" está em modo manual
(workflow_dispatch) para evitar falhas enquanto não houver
credenciais. Depois de configurar os secrets, publique assim:

1. Fornecedor de alojamento: servidor FTP, utilizador, password,
   pasta remota (ex: /public_html/site/)

2. GitHub → Stoffus-pt/site → Settings → Secrets and variables → Actions
   Secrets obrigatórios:
   | Nome            | Exemplo              |
   |-----------------|----------------------|
   | FTP_SERVER      | ftp.stoffus.pt       |
   | FTP_USERNAME    | o_vosso_user         |
   | FTP_PASSWORD    | a_vossa_password     |
   | FTP_REMOTE_DIR  | /public_html/site/   |
   Opcional: FTP_PORT (default 21)

3. GitHub → Actions → "Publicar site (FTP)" → Run workflow

4. Confirmar:
   https://stoffus.pt/site/index.html
   CMS: https://stoffus.pt/site/cms/  (ou /cms/ conforme pasta)

NÃO enviar cms/config.php se já existir no servidor com password.

Manual (FileZilla / cPanel): enviar o conteúdo de "site/",
excluindo cms/config.php, node_modules, .git, backups.


Fluxo recomendado
-----------------
1. Alterar localmente → push → validar em GitHub Pages
2. Quando estável → Actions → Publicar site (FTP)
3. Validar em stoffus.pt (CMS, formulários, fotos)


Tecidos
-------
Fonte Studio3D: npm run fabrics:sync (em site/)
Overrides de capas/textos/visibilidade: CMS → separador Tecidos
  (grava data/fabrics-site.json e tenta regenerar fabrics.json)


Notas
-----
- Upload de fotos no CMS online exige Node.js no servidor
  (raro em alojamento partilhado). Alternativa: fotos locais +
  git push + deploy FTP.
- O repositório GitHub é backup + histórico; o site “vivo”
  está no alojamento stoffus.pt.
