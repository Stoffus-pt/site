Publicar o site Stoffus online
==============================

FASE DE TESTES — ver no GitHub (sem FTP)
----------------------------------------
Cada push para main actualiza automaticamente a pré-visualização:

  https://stoffus-pt.github.io/site/

1. GitHub → repositório Stoffus-pt/site → Settings → Pages
2. Em "Build and deployment", escolha Source: **GitHub Actions**
3. Faça push para main (ou Actions → "Pré-visualização (GitHub Pages)" → Run)

Inclui: páginas HTML, catálogo, fotos, modelos (models.json).
NÃO inclui: CMS (PHP), upload de fotos pelo browser.

Quando quiser editar textos pelo CMS online, use o alojamento stoffus.pt
(secção abaixo).


PRODUÇÃO — stoffus.pt (com CMS PHP)
-----------------------------------
Para VER o site no alojamento real precisa de publicar num servidor com PHP.

OPÇÃO A — Automático (recomendado)
----------------------------------
Cada vez que fizer push para o GitHub, o site actualiza-se sozinho.

1. Peça ao vosso fornecedor de alojamento (ou quem gere o stoffus.pt):
   - Servidor FTP (ex: ftp.stoffus.pt)
   - Utilizador e palavra-passe FTP
   - Pasta de destino (ex: /public_html/site/ ou /public_html/novo/)

2. No GitHub: https://github.com/Stoffus-pt/site
   → Settings → Secrets and variables → Actions → New repository secret

   Crie estes segredos:
   | Nome            | Exemplo              |
   |-----------------|----------------------|
   | FTP_SERVER      | ftp.stoffus.pt       |
   | FTP_USERNAME    | o_vosso_user         |
   | FTP_PASSWORD    | a_vossa_password     |
   | FTP_REMOTE_DIR  | /public_html/site/   |

3. GitHub → Actions → "Publicar site (FTP)" → Run workflow
   (ou faça push para main)

4. Abra no browser:
   https://stoffus.pt/site/index.html
   (ou o URL que corresponda à pasta FTP_REMOTE_DIR)

CMS online: https://stoffus.pt/cms/
(na primeira visita, crie utilizador e palavra-passe)


OPÇÃO B — Manual (FileZilla / cPanel)
-------------------------------------
1. Ligue por FTP ao alojamento
2. Envie TODO o conteúdo da pasta "site" para a pasta no servidor
3. NÃO envie cms/config.php se já existir no servidor com password
4. PHP 8+ tem de estar activo no alojamento


Versão de teste (não substituir o site actual)
----------------------------------------------
Peça um subdomínio, por exemplo: novo.stoffus.pt
Aponte para uma pasta vazia e use FTP_REMOTE_DIR para essa pasta.
Assim testa o site novo sem afectar o WordPress/site actual.


Notas
-----
- Upload de fotos no CMS online exige Node.js no servidor (raro em alojamento
  partilhado). Alternativa: carregar fotos localmente, git push, deploy FTP.
- O repositório GitHub continua a ser backup + histórico; o site "vivo" está
  no alojamento stoffus.pt.
